git => Version control systems

github => will track those changes and 'u' refers untracked files & M for Modified

git commands:

  1. git --version
  2. git init
    • git init - To initialze the git in the project
  3. git stauts
    • git status means, it will show what is tracked and what is untracked
  4. git add (file path name)
    • Example: git add gitandgithub\intro.html
  5. git commit
    • git commit -m "any thing we can write here"
  6. If u r a first user then, add email and username
    • git config --global user.email "bdharshini61997@gmail.com"
    • git config user.name "Dharshini Balasundaram1997"
  7. git add .
    • add . this will add all the files
  8. git commit -m "your messaage"
    • git commit -m "any thing we can write here"
    • m => means message
    • Example: git commit -m "added coded"
  9. git log
    • it will show who has done what and it will show the commit messages
  10. git reset --hard (enter the commit number)
    • example: git reset --hard 3f0ec6db2314e43d3d1a08fe786bbec4ffe35c22
      hard means => everything it will clear
  11. git reset --soft (enter the previous commit number)
    • Example: git reset --soft 629152322a05bea0eb980a385750fe8019cf8fb8
    • => u can go to previous commits without deleting the present code by using soft
